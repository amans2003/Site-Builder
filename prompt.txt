# Project Prompt: MERN-Based Drag-and-Drop Website Builder (WordPress/Wix-style)

## Overview
Build a full-stack web application using the **MERN stack (MongoDB, Express, React, Node.js)** that allows non-technical users to visually build informative websites, similar to WordPress/Wix/Webflow. Users should be able to construct a full website using pre-built sections, edit content live, preview responsively, optionally inject custom code, and finally **export/download the finished site as static HTML/CSS/JS files** ready to be uploaded to a cPanel hosting account (no backend dependency required for the exported site).

---

## Core Features

### 1. Section-Based Page Builder
- Sidebar panel listing pre-built sections: Navbar, Hero, About, Services, Gallery, Testimonials, Pricing, Contact Form, Footer, etc.
- Clicking a section adds it to the canvas at the desired position (click-to-insert, plus optional drag-and-drop reordering).
- Each section is a reusable component driven by a JSON schema (not hardcoded HTML), so it can be rendered both in the live editor and in the final static export.

### 2. Contextual Editing (per-section settings panel)
- When a user selects/hovers a section, show a floating toolbar or side panel with edit options relevant only to that section type.
- Example — Navbar section should allow editing:
  - Logo (upload/replace, position: left/center)
  - Menu items (add/remove/reorder links)
  - Profile/account icon (toggle visibility, position)
  - Background color, sticky/fixed toggle
- Every section type needs its own editable field schema (generate settings UI dynamically from this schema rather than hardcoding a form per section).

### 3. Live Content Editing
- Inline text editing (click text to edit directly on canvas)
- Image upload/replace within sections (drag-drop or file picker)
- Color pickers, spacing/padding controls, font selection where relevant

### 4. Custom Code Block
- Add a "Custom Code" section type that opens a code editor (use Monaco Editor) allowing the user to write raw HTML/CSS/JS.
- Render this safely — sanitize HTML (DOMPurify) and isolate JS execution using a sandboxed `<iframe>` to avoid breaking the main builder.

### 5. Responsive Preview Toggle
- Toolbar buttons to preview the canvas at Desktop / Tablet / Mobile widths.
- Canvas container width changes accordingly (375px / 768px / 100%), and section CSS must use real responsive rules (media queries), not just visual scaling.

### 6. Project Persistence
- User authentication (JWT-based) — signup/login
- Save/load multiple website projects per user
- Each project stored as a structured JSON document (not raw HTML) representing the full page/section tree

### 7. Export & Download for cPanel Hosting
- Convert the JSON project structure into clean static HTML/CSS/JS files
- Bundle all uploaded images into an `/assets` folder with correct relative paths
- Package everything into a `.zip` file (use `archiver` on the backend) for the user to download
- The exported site must run as pure static files — no Node/Express server dependency — so it can be uploaded directly into `public_html` via cPanel File Manager or FTP

---

## Suggested Tech Stack
- **Frontend:** React, `@dnd-kit` or `react-dnd` (drag-and-drop), Monaco Editor (code block), TailwindCSS or CSS Modules for styling
- **Backend:** Node.js, Express, JWT auth, Multer (image uploads), Archiver (zip export)
- **Database:** MongoDB with Mongoose — store each project as a JSON tree of pages → sections → props
- **Optional accelerator:** Consider building on top of **GrapesJS** (open-source visual builder) instead of a fully custom drag-and-drop engine — it already provides section blocks, a trait/settings manager, a code editor plugin, and responsive device preview out of the box. Wrap MERN around it for auth, project storage, and the export/zip pipeline.

```json
{
  "projectId": "abc123",
  "userId": "user_456",
  "pages": [
    {
      "name": "home",
      "sections": [
        {
          "type": "navbar",
          "props": {
            "logo": { "src": "/uploads/logo.png", "position": "left" },
            "menu": ["Home", "About", "Contact"],
            "profileIcon": { "enabled": true, "position": "right" },
            "sticky": true
          }
        },
        {
          "type": "hero",
          "props": {
            "title": "Welcome to My Business",
            "subtitle": "We build great things",
            "bgImage": "/uploads/hero-bg.jpg"
          }
        }
      ]
    }
  ]
}
```

## Deliverables Requested
1. Project folder structure for a MERN app with this architecture
2. Section/block registry pattern (schema-driven components, not hardcoded per section)
3. Dynamic settings-panel generator based on a section's field schema
4. Working drag-and-drop / click-to-insert canvas
5. Responsive preview toggle (desktop/tablet/mobile)
6. Custom code block with Monaco Editor + sandboxed rendering
7. Authentication + MongoDB project save/load
8. Export pipeline: JSON → static HTML/CSS/JS → zipped download, cPanel-ready

## Development Approach
Please build this incrementally:
1. Start with the data model + a basic renderer that turns JSON into a live page
2. Add 3–4 core sections (Navbar, Hero, Text, Footer) with editable schemas
3. Build the settings panel generator
4. Add drag-and-drop / click-to-insert
5. Add responsive preview toggle
6. Add auth + save/load to MongoDB
7. Add the custom code block
8. Build the export-to-zip pipeline last, once the section renderer is stable
